### 問題箇所

**(1) ショートポーリングのコード例に「処理の重なり」への注意がない**

> ```javascript
> // 5秒ごとに更新を確認する
> setInterval(poll, 5000);
> ```

`setInterval` は前回の `poll()` の完了を待ちません。
ネットワーク遅延で `fetch` が5秒以上かかると、リクエストが重なって飛びます。
実務では `setTimeout` の再帰で「前回完了後に次を予約」するのが定石です。
最小コードとしては許容範囲ですが、`:::message` で一言注意があると正確性が上がります。

```javascript
async function poll() {
  const response = await fetch('/api/updates');
  console.log(await response.json());
  setTimeout(poll, 5000); // 完了してから次を予約する
}
poll();
```

**(2) ロングポーリングのコード例にバックオフがない**

> ```javascript
>   } catch (error) {
>     console.error(error);
>   } finally {
>     // レスポンスを受け取ったら、すぐ次の問い合わせを始める
>     longPoll();
>   }
> ```

サーバー障害でリクエストが即失敗する状況だと、`finally` で即再帰するため、
リクエストが高速ループで暴走します（サーバーへの追い打ちにもなる）。
エラー時は待機（バックオフ）を挟むのが定石です。
このコードが「最小で動く」のは正しいですが、注意書きが1行あると親切です。
