## 1. 内容の正確性: 23/25

### Good

- 「React Hook Form は非制御コンポーネントを使う」という核心が正確。これが再描画の少なさに直結する説明も筋が通っている。
- 制御／非制御の定義（状態で管理 vs DOM が保持）が簡潔かつ正しい。
- `register` で登録 → `handleSubmit` で値をまとめて取得、という流れが実際の API の役割と一致。
- Formik が制御コンポーネント寄りで再描画が起きやすい、という比較も妥当。
- Zod などのスキーマ連携への言及も正確。

### 問題箇所

**(1) 「入力中は再描画なし」は条件付きで、例外に触れたい**

> D[入力値は DOM が保持<br/>＝入力中は再描画なし]

基本はそのとおりですが、例外があります。
`watch` で入力値を監視したり、`Controller` で UI ライブラリ（MUI など）と繋ぐと、再描画は発生します。
「基本は再描画を抑えられる。ただし `watch` などで値を監視する場合は別」と一言あると、より正確で、実装時の誤解を防げます。

**(2) バージョン・前提の明記がない（軽微）**

React Hook Form は v7 で API が整理されました。
「本記事は v7 を前提」と冒頭か脚注に1行あると、読者が手元のバージョンとズレないか確認できます。概念記事なので必須ではありません。
