- FiberがReact 16から、並行レンダリングがReact 18からという時系列が正確。

### 問題箇所

**(1) JSXの変換先が古い**

> JSXはビルド時に`React.createElement()`の呼び出しへ変換され、最終的に次のようなオブジェクトになります。

React 17（2020年）以降、デフォルトのJSX変換は **automatic runtime**（`react/jsx-runtime` の `_jsx` / `_jsxs`）に変わっています。2026年の記事で「`React.createElement` に変換される」と断定すると、やや古い説明です。

修正案: 「（React 16までは）`React.createElement()` に変換されていました。React 17以降は `jsx` ランタイムに変わりましたが、**生成される要素オブジェクトの考え方は同じ**です」と一言添えると正確になります。

**(2) 仮想DOM＝「軽量なコピー」「高速」の単純化**

> 仮想DOMとは、**実際のDOMを模した軽量なJavaScriptオブジェクト**です。

> これはただのJSオブジェクトなので、生成や比較が高速です。

この説明自体は誤りではありませんが、「仮想DOMは実DOM操作より速いための仕組み」という誤解を生みやすい表現です。React開発者自身が「仮想DOMの主目的はパフォーマンスではなく、宣言的に書くための仕組み」と繰り返し補足しています。

修正案: `:::message` で「仮想DOMは『直接DOMを触るより必ず速い』わけではありません。本質は**宣言的に書きつつ、実DOMへの操作回数を抑えられる**点です」と補足すると、踏み込んだ正確さが出ます。
